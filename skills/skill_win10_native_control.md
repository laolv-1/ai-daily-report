# skill_win10_native_control.md

## 适用场景

当主公要求在 **不引入第三方高风险桌面技能** 的前提下，为 Win10 节点建立最小可控、无网络外发的本地桌面控制能力时使用。

目标仅限：

- 窗口激活 / 聚焦
- 安全文本输入
- 记事本等基础软件的最小演示

严禁内置：

- 任意外网请求
- 遥测上报
- 自动下载执行
- 隐蔽驻留
- 注册表持久化
- 未授权跨网通信

---

## 设计原则

1. **纯本地执行**：所有动作只在 Win10 本机完成。
2. **最小能力面**：初版只允许窗口查找、激活、文本输入。
3. **无网络栈**：脚本中不引入 `requests`、`socket`、`urllib`、`httpx` 等网络模块。
4. **可审计**：优先使用 Python 标准库 + 极少数可解释依赖。
5. **可回退**：每次动作前后都能靠截图/窗口标题/进程名做核验。

---

## 推荐实现栈

### 方案 A：PowerShell + AppActivate + SendKeys（最轻）

适合：
- 记事本
- 简单输入框
- 低复杂度演示

优点：
- Win10 原生
- 无需额外 Python GUI 自动化包

风险：
- 稳定性一般
- 对复杂输入法和 IDE 兼容性较弱

### 方案 B：Python + ctypes（推荐）

用途：
- 枚举窗口
- 查找标题
- 激活窗口
- 向前台窗口发送安全文本

优点：
- 无需引入整套重型第三方桌面框架
- 能力边界清晰
- 更容易做源码审计

### 方案 C：Python + pywinauto（仅在主公批准后）

用途：
- 复杂 UI 控件识别
- 菜单/按钮/编辑框级操作

风险：
- 依赖更重
- 后续若引入，必须先独立审计依赖树

---

## 最小原型建议

### 1. 激活记事本

- 启动 `notepad.exe`
- 轮询窗口标题是否出现
- 将窗口置前

### 2. 输入文本

- 将窗口切为前台
- 仅允许输入安全白名单文本
- 不执行热键宏
- 不发送系统级危险组合键

### 3. 核验

- 读取窗口标题
- 必要时截图
- 返回“目标窗口标题 + 已输入字符数”作为回执

---

## 安全红线

以下能力默认永久禁用，除非主公单独书面批准：

- 浏览器密码页输入
- 系统设置修改
- 注册表写入
- 后台静默下载
- 任意进程注入
- 键盘记录
- 屏幕常驻窃取
- 任何网络传输

---

## 当前状态

- 第三方候选 `windows-ui-automation`、`desktop-control-win` 已被标为 suspicious，禁止盲装。
- 因此当前 Win10 桌面控制路线改为：**只借思路，不纳毒包；先写自研最小纯净技能。**
- 后续实装时，优先从 `PowerShell AppActivate + SendKeys` 或 `Python ctypes` 起步。
