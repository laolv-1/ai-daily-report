# workflow-cyber-talkshow-generator blueprint

## 1. 目标

把一句话指令，例如：

- `生成今日份虾小侠短视频音频`

扩展成完整自动流水线：

1. 抓素材
2. 汇总要点
3. 用 `lck/gpt-5.4` 生成适合口播的短文案
4. 调度 Win10 官方 Relay 浏览器
5. 打开免费 TTS 站
6. 注入文本并点击生成
7. 等待音频下载完成
8. 归档文件
9. 发回主公

---

## 2. 推荐的白嫖 TTS 工作流

### 主路

站点优先尝试：

1. `https://ttsmaker.com/`

理由：

- 常见公开站点
- 中文可用概率高
- 页面通常有明确文本输入框与生成/下载入口
- 不依赖本地安装额外客户端

### 浏览器动作骨架

1. `browser tabs` / `browser open`
2. `browser snapshot` 获取输入框、语音选择器、生成按钮、下载按钮
3. `browser act/fill/type/click` 完成文案注入与生成
4. `browser wait` 或下载等待逻辑等待生成完成
5. `browser waitfordownload` 或页面下载链接点击

### DOM / UI 定位逻辑

优先按以下顺序定位：

1. 文本框：`textarea` / `combobox` / `textbox`
2. 生成按钮：按钮文案如“转换”“生成”“开始转换”“Convert”“Synthesize”
3. 下载按钮：按钮文案如“下载”“Download”
4. 若页面文案改版，则重新 `snapshot` 后更新定位词

不要把定位逻辑写死成单一 CSS 选择器；优先用可读的按钮名与 aria/role。

## 3. 下载与归档

### 推荐落盘目录

Win10：

- `C:\Users\Administrator\Downloads\openclaw_tts\`

主脑归档：

- `/root/.openclaw/workspace/artifacts/tts/`

### 文件命名建议

- `xiaxiaoxia-YYYYMMDD-topic.mp3`

### 完成判据

至少满足一个：

1. 浏览器下载事件成功
2. 目标目录出现新音频文件
3. 文件大小大于 0 且扩展名正确（`.mp3` / `.wav`）

---

## 4. 自主纠错 / Retry 设计

### Retry 上限

- 同一任务最多 3 次

### 第 1 次失败

- 重新抓页面结构
- 替换定位词
- 重试当前站点

### 第 2 次失败

- 切备用交互方式：原生窗口控件 / 键盘输入 / 粘贴
- 重试当前站点

### 第 3 次失败

- 切备用 TTS 站点
- 再跑一轮完整链路

### 仍失败

向主公发送一句人类可读结论，例如：

- `免费 TTS 站点出现风控/页面结构大改，当前自动生成链被阻断，需要换站或人工首轮校准。`

不要直接抛 traceback。

---

## 5. 宏技能输入/输出约定

### 输入

- 主题
- 语气（毒舌/冷静/锐评）
- 时长（如 30 秒 / 60 秒）
- 是否需要口播标题
- 是否自动回传音频

### 输出

- 文案文件路径
- 音频文件路径
- 归档状态
- 发回主公的消息文本

---

## 6. 架构纪律

- 浏览器主路：官方 Relay
- 桌面兜底：Native Control
- 模型主路：`lck/gpt-5.4`
- 对主公输出必须简洁，不要噪音式调试日志
- 除非连续 3 次失败，否则不向主公求助
